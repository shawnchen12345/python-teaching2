# 零基础大模型应用实战课程 (8小时速成版)

**核心调整**: 去掉了复杂的训练/微调环节，专注于**如何把现成的强大模型用起来**。
**目标**: 学员能够独立加载开源大模型 (Llama-3/Qwen)，并基于它开发出解决实际问题的应用 (如文档问答、智能助手)。

**前置要求**:
*   会基本的 Python。
*   电脑显卡显存 8G+ (或者使用 Colab/AutoDL)。

---

## 📅 课程大纲

### 第一阶段：环境搭建与模型部署 (2.5小时) —— "把大象请回家"

**目标**: 让开源模型在本地流畅运行，并提供 API 接口。

1.  **环境配置 (45分钟)**
    *   **快速上手**: 使用 `conda` 创建干净的 Python 环境。
    *   **显卡驱动**: 确保 `nvidia-smi` 正常工作，安装 PyTorch (GPU版)。
    *   **核心库**: `transformers`, `accelerate`, `bitsandbytes` (量化神器)。

2.  **加载开源模型 (1小时)**
    *   **选择模型**: 介绍 Hugging Face / ModelScope。推荐 `Qwen1.5-7B-Chat-Int4` (量化版，显存占用极小，速度快)。
    *   **代码实操**:
        *   使用 `AutoModelForCausalLM` 加载模型。
        *   实现一个简单的 Python 脚本，在终端和 AI 对话。
        *   **关键点**: 演示 fp16 (半精度) vs int4 (量化) 的区别 —— 速度起飞，显存减半。

3.  **API 服务化 (45分钟)**
    *   **为什么要 API?**: 为了让其他程序 (如网页、Excel) 能调用模型。
    *   **工具**: 使用 `FastAPI` 封装模型接口。
    *   **实操**: 写一个 `api.py`，启动后，可以通过 `POST` 请求发送问题并获得回答。

---

### 第二阶段：提示词工程与应用开发 (2.5小时) —— "教大象办事"

**目标**: 不改模型参数，通过 Prompt Engineering 让模型变聪明。

4.  **Prompt Engineering 进阶 (1小时)**
    *   **System Prompt**: 赋予模型人格 (如: "你是一个资深 Python 程序员")。
    *   **One-Shot / Few-Shot**: 给模型看几个例子，让它照猫画虎。
    *   **CoT (思维链)**: 即使模型没训练过数学，只要让它 "一步步思考"，它就能做对。

5.  **结构化输出 (45分钟)**
    *   **痛点**: 模型废话太多。我只想要 JSON 格式！
    *   **实操**: 设计 Prompt，强制让模型输出 `{"sentiment": "positive", "score": 0.9}` 格式，方便代码处理。

6.  **LangChain 初体验 (45分钟)**
    *   **概念**: "胶水"。把模型和外部工具粘在一起。
    *   **实操**: 使用 LangChain 构建一个简单的流程链: 接收用户输入 -> 格式化 Prompt -> 调用模型 -> 解析输出。

---

### 第三阶段：打造超级应用 (3小时) —— "给大象装上外脑"

**目标**: 这是课程的高潮。构建一个 **RAG (检索增强生成)** 系统，让 AI 能回答它没学过的私有知识。

7.  **RAG 原理与向量数据库 (1小时)**
    *   **痛点**: 模型不知道刚才上传的 PDF 内容。
    *   **解决方案**: 把文档切块 -> 变成向量 (Embedding) -> 存入数据库 (ChromaDB/Faiss)。
    *   **比喻**: 就像考试时允许翻书。模型先去数据库里查相关段落，再回答问题。

8.  **实战：本地文档问答助手 (1.5小时)**
    *   **架构**:
        1.  用户上传 PDF/TXT。
        2.  系统自动切分并向量化。
        3.  用户提问 -> 检索相关片段 -> 拼接到 Prompt 中 -> 模型回答。
    *   **实操**: 写一个 Python 脚本实现上述流程。

9.  **Streamlit 前端展示 (30分钟)**
    *   **整合**: 把 RAG 系统的后台逻辑，接到 Streamlit 的聊天界面 (`st.chat_message`) 上。
    *   **成果**: 一个可以上传私有文档、并基于文档问答的 Web App。

## 📝 大作业

**题目**: 开发一个 "智能会议纪要助手"。
**要求**:
1.  用户输入一段会议录音转写的乱糟糟的文本。
2.  系统自动总结摘要、列出待办事项 (To-Do List)。
3.  输出格式必须是 Markdown 表格。
4.  使用 Streamlit 展示。
